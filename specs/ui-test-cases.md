## 全站基础与非功能性 UI 测试 (NFR)

1. **响应式及缩放 (Responsiveness)**：在 1920x1080 与 1366x768 下查看所有表格数据（项目详情、资源看板），均能提供充足的横向滚动条，操作列（右侧 `fixed` 列）无论怎么滚动均始终停靠最右。
2. **防跳动 (Layout Shift)**：切换顶部 Tabs 或左侧树菜单时，页面无明显重排（Reflow）闪烁。Loading 占位符必须同尺寸过渡。
3. **按钮防连击 (Debounce/Loading)**：表单中的所有”保存/创建”按钮，在点击后会立刻 Disable 并呈现菊花（Loading）动画，直至接口返回才恢复，彻底杜绝短时间内并发两次一样的 POST 请求。
4. **可访问性和容错 (Accessibility & Error Boundary)**：如果后端 502/504，所有页面能捕获并在内部抛出优雅的”网络开小差”错误通知，而不是整面 React 白屏报错。
5. **表头禁止换行 (Header No-Wrap)**：全站所有表格的表头文字在 1920x1080 分辨率下不允许出现换行。若表头文字被挤压换行，说明列宽不足，应调宽该列的 `width` 直到表头文字恢复单行显示。覆盖页面：项目列表、项目详情（活动列表）、产品列表、周报汇总、资源负荷、系统管理各子页。
6. **表头与数据列对齐 (Column Alignment)**：全站所有表格中，表头文字的左边缘必须与对应数据行内容的左边缘精确对齐。带排序功能的列需通过 CSS 重置 Arco 默认的 sorter 内边距（`padding: 0 16px`），使其与非排序列的 `padding-left: 16px` 一致。紧凑表格（`.compact-table`）中 sorter 内边距为 `0 6px`。
7. **紧凑/标准表格分层 (Table Density)**：项目列表等列数少的页面使用 Arco 默认间距（标准表格），活动列表等多列页面通过 `.compact-table` wrapper 使用紧凑间距（cell padding `6px`）。
8. **固定列滚动条对齐 (Fixed Column Scrollbar)**：使用 `fixed: 'right'` 的操作列，需确保表头和表体在有/无垂直滚动条时列边界对齐。通过 `.arco-table-header { overflow-y: scroll }` 配合透明滚动条预留与 body 相同的滚动条宽度。
