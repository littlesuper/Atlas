# Atlas 需求文档

本目录用于存放 Atlas 新需求的结构化规约。Week 3 起，所有新需求必须先在这里形成需求文档，再进入开发。

## 使用方式

1. 复制 `REQUIREMENT_TEMPLATE.md`。
2. 命名为 `REQ-XXX-需求名称.md`，例如 `REQ-024-project-filter-empty-state.md`。
3. 填写所有章节，不允许保留空白、`待定`、`以后再说`。
4. 使用 `docs/prompts/01-需求澄清提示词.md` 审查需求。
5. 修订到无阻断问题后，才允许进入代码生成。

## 审核规则

AI 代码守护人审核需求文档时，至少检查：

- 业务目标、用户角色、使用场景是否明确。
- 输入、输出、业务规则、异常场景是否完整。
- 验收标准是否可测试，是否有 Gherkin 场景。
- 是否涉及权限、审计、登录、企业微信、AI 配置、数据库迁移、生产配置或密钥。
- 如涉及高风险事项，必须暂停并单独决策。

## 状态约定

需求文档的 `状态` 字段使用以下值：

- `草稿`：业务方正在补充。
- `评审中`：AI 需求审查或人工审查进行中。
- `已批准`：可以进入实现。
- `开发中`：已有实现分支或 PR。
- `已完成`：已合并并完成验证。
