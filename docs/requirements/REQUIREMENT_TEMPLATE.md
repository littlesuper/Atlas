## 5. 异常场景(必填,**这是最常被忽略的部分**)

### 5.1 输入异常

| 场景 | 系统行为 |
|------|---------|
| keyword包含SQL注入字符 | 安全转义,正常搜索 |
| keyword超过100字符 | 返回400,提示"关键词过长" |
| startDate格式错误 | 返回400,提示"日期格式错误" |
| endDate早于startDate | 返回400,提示"结束日期必须晚于开始日期" |
| page为负数 | 返回400,提示"页码无效" |
| pageSize超过100 | 返回400,提示"单页不能超过100条" |

### 5.2 业务异常

| 场景 | 系统行为 |
|------|---------|
| 用户未登录 | 返回401,跳转登录页 |
| 用户Token过期 | 返回401,前端自动刷新Token或跳登录 |
| 用户被封禁 | 返回403,显示"账号已被禁用" |
| 用户没有任何订单 | 返回空数组,前端显示"暂无订单"插画 |
| 搜索结果为空 | 返回空数组,前端显示"未找到匹配订单" |
| 请求超过频率限制 | 返回429,提示"请求过于频繁,请稍候" |

### 5.3 系统异常

| 场景 | 系统行为 |
|------|---------|
| 数据库连接失败 | 返回500,记录错误日志,前端显示通用错误页 |
| 查询超时(>5s) | 返回504,记录慢查询日志 |
| 第三方服务故障(如有) | 优雅降级:展示缓存数据 |
| 服务器内存不足 | 触发告警,自动扩容 |

### 5.4 并发场景

| 场景 | 系统行为 |
|------|---------|
| 用户快速翻页 | 上一个请求未完成时,新请求自动取消上一个 |
| 用户同时打开多个标签页 | 每个标签页独立工作,不冲突 |
| 用户在不同设备登录 | 互不影响 |

## 6. 验收标准(必填,可测试)

### 6.1 功能验收(Gherkin格式)

```gherkin
Feature: 订单列表查询

  Background:
    Given 用户 "test@example.com" 已登录
    And 用户有以下订单:
      | 订单号        | 状态  | 金额 | 创建时间   |
      | 20240101-001 | paid  | 100  | 2024-01-01 |
      | 20240115-002 | shipped | 200 | 2024-01-15 |
      | 20240201-003 | done  | 300  | 2024-02-01 |

  Scenario: 默认查看订单列表
    When 用户进入"我的订单"页面
    Then 应显示3条订单
    And 订单按创建时间倒序排列
    And 第一条订单是 "20240201-003"

  Scenario: 按关键词搜索
    When 用户输入关键词 "20240115"
    Then 应只显示1条订单
    And 显示的是 "20240115-002"

  Scenario: 按状态筛选
    When 用户选择状态 "已完成"
    Then 应只显示1条订单
    And 显示的是 "20240201-003"

  Scenario: 没有匹配结果
    When 用户输入关键词 "不存在的订单号"
    Then 应显示空状态插画
    And 显示文字 "未找到匹配订单"

  Scenario: 未登录访问
    Given 用户未登录
    When 用户访问 "/orders" 页面
    Then 应跳转到登录页面
    And 登录后跳回 "/orders" 页面
```
