# Atlas AI 提示词库

本目录存放 Atlas 团队日常使用的 7 个标准提示词。它们来自 `atlas-quality-system/prompts/`，复制到仓库后可在 PR、需求文档和团队 Wiki 中直接链接。

## 快捷入口

| 提示词 | 使用场景 | 产物 |
| --- | --- | --- |
| `01-需求澄清提示词.md` | 写代码前审查需求 | 需求问题清单、修订建议 |
| `02-代码生成提示词.md` | 让 AI 生成实现 | 代码改动方案、文件清单 |
| `03-代码审查提示词.md` | PR 前或 PR 中做 AI 审查 | 风险清单、阻断问题 |
| `04-测试生成提示词.md` | 为需求或改动生成测试 | Vitest / Playwright / Supertest 测试建议 |
| `05-Bug诊断提示词.md` | 排查线上或测试失败问题 | 复现路径、根因假设、修复计划 |
| `06-架构守护提示词.md` | 评估架构、边界和长期风险 | 架构风险与约束建议 |
| `07-上线前检查提示词.md` | 发布前检查 | 上线风险、回滚与观察项 |

## 使用规则

1. 每个新需求至少使用 `01-需求澄清提示词.md`、`02-代码生成提示词.md`、`04-测试生成提示词.md`。
2. 每个 PR 合并前至少使用 `03-代码审查提示词.md` 做一次 AI 自审。
3. 涉及上线、发布、生产环境或回滚时，必须使用 `07-上线前检查提示词.md`。
4. 涉及模块边界、数据模型、权限、AI 配置或跨系统影响时，必须使用 `06-架构守护提示词.md`。
5. 遇到 bug、CI 失败、E2E flaky 或线上异常时，先使用 `05-Bug诊断提示词.md`，不要靠猜测修。
