# Week 3 Day 5 流程回顾执行表

本文件用于执行 Week 3 Day 5：第一次流程回顾。它是复盘材料，不代表复盘已经完成。

## 目标

- 汇总 Day 1-2 培训和 Day 3-4 模板/提示词试用中的痛点。

## 简化候选

只有满足以下条件的小修复，才允许走简化流程：

- 不涉及密钥、生产配置、数据库迁移或数据修复。
- 不改变权限、登录、审计、企业微信、AI 配置。
- 不改变已有 API 契约或数据模型。
- 可以用一个明确测试或一个明确手工验证步骤覆盖。

| 可简化步骤 | 适用场景 | 保留底线 |
| --- | --- | --- |
| 需求文档 | 纯文案、样式微调、无行为变化的小修复 | 必须写清问题、预期行为、验证方式 |
| Gherkin 场景 | 非业务行为改动 | 必须有可复现/可验证描述 |
| 完整提示词链路 | 低风险小修复 | 仍需使用代码审查提示词或人工 review |

## 加强候选

以下情况不能简化，必要时还要增加审查：

- 新增或修改 Prisma schema、迁移、seed、生产数据脚本。
- 修改认证、权限、角色、审计、企业微信、AI 配置。
- 修改上传、富文本、HTML 渲染、Excel 导入导出等高风险输入路径。
- 修改 CI、分支保护、发布、回滚、生产配置。

| 需加强步骤 | 触发条件 | 加强方式 |
| --- | --- | --- |
| 需求审查 | 权限/数据/生产影响 | AI 守护人必须签字 |
| 架构审查 | 跨模块或模型变更 | 使用 `docs/prompts/06-架构守护提示词.md` |
| 上线检查 | 已部署服务影响 | 使用 `docs/prompts/07-上线前检查提示词.md` |
