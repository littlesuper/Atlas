# Week 3 Day 1-2 流程培训执行包

本文件用于执行 Week 3 Day 1-2：流程培训与手把手实战。它是执行材料，不代表培训已经完成；培训完成后需要在本文末尾填写记录。

## 培训目标

- 团队能说出 Atlas AI 协作开发的 14 个关键步骤。
- 团队知道什么时候使用需求模板、提示词、检查清单。
- 至少选择 1 个真实小需求，按流程从需求澄清走到 PR。
- AI 代码守护人记录卡点，作为 Day 5 流程回顾输入。

| 时间 | 内容 | 负责人 | 输出 |

| 110-120 分钟 | 收集问题和流程卡点 | 全员 | Day 5 回顾输入 |

## 培训前准备

- 确认所有参与者能访问 GitHub 仓库。
- 确认本地已安装 Node.js `>=20.19.0`。
- 确认本地已安装 pre-commit，或知道如何运行 `python3 -m pre_commit run --all-files`。
- 打开本文档和 `docs/process/atlas-ai-development-workflow.md`。
- 准备 1 个真实小需求，要求满足：
  - 不涉及生产密钥、生产配置或数据库迁移。
  - 不改变权限模型、登录流程、企业微信配置或 AI 配置。
  - 可以在 1 天内完成需求澄清、实现、测试和 PR。

## 推荐实战需求类型

优先选择低风险、可测试的小需求：

- 列表筛选、排序、空状态或错误提示优化。
- 管理页表单校验文案补充。
- 已有页面的导出、搜索、分页边界修复。
- 后端只读查询接口的小范围字段补充。

不建议作为首次实战：

- Prisma schema 变更。
- 权限、角色、审计、登录、企业微信、AI 配置。
- 生产数据修复。
- 大规模 UI 重构。
