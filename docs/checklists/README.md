# Atlas 质量检查清单

本目录存放 Week 8 Day 3 优化后的 Atlas 检查清单。来源是 `atlas-quality-system/checklists/`，但已按 Atlas 当前技术栈和前 7 周落地经验改写。

## 清单入口

| 文件 | 使用时机 | 主要产物 |
| --- | --- | --- |
| `每日开发检查清单.md` | 每个工作日、每个任务开始前 | 今日任务边界、健康状态、风险提醒 |
| `提交代码前检查清单.md` | commit / push / PR 前 | 本地验证记录、变更范围确认 |
| `上线前检查清单.md` | 生产发布前 24-48 小时和发布前 1 小时 | Go/No-Go 结论、回滚方案 |
| `生产事故应急清单.md` | staging / production 出现异常时 | 事故等级、止血动作、复盘记录 |
| `月度质量审计清单.md` | 每月质量复盘 | 月报、Top 改进项、下月目标 |

## 使用原则

1. 清单是最低标准，不是形式主义。明显不适用的条目要写明原因，不要假装完成。
2. 涉及密钥、生产配置、数据库迁移、真实回滚、staging/production 故障演练时，先停下来确认负责人授权。
3. 文档变更可以走简化验证，但仍需 PR 和 GitHub required checks。
4. 每月审计时根据实际拦截效果更新清单；没人在看的条目应删除或自动化。
