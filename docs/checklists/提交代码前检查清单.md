# 提交代码前检查清单

> 使用频率：每次 commit、push、创建 PR 前
> 目标：不把未验证的 AI 代码带入主干

## 变更范围确认

- [ ] `git diff` / Source Control 面板已逐文件看过。
- [ ] 每个改动都能解释原因，没有 AI 顺手改的不相关文件。
- [ ] 没有提交 `.env`、数据库文件、日志、临时文件、个人 IDE 配置或大文件。
- [ ] 没有新增硬编码 token、JWT secret、Unleash 后端 token、生产 URL。
- [ ] 没有新增生产代码中的 `TODO/FIXME`、注释掉的死代码、调试输出。

## Atlas 必要验证

按改动范围选择最小但足够的验证集：

```bash
npm run lint
npm run test
npm run test:coverage
npm run coverage:core
npm run build
```

涉及 Prisma、server 测试、integration test 或新 worktree 时：

```bash
cd server
npx prisma generate
```

涉及 E2E、视觉、无障碍或核心用户路径时：

```bash
npm run test:e2e:core
npm run test:e2e:a11y
npm run test:e2e:visual
```

## 质量门禁

- [ ] `npm run lint` 无 error；不要新增 warning。
- [ ] `npm run audit` 无 high / critical 漏洞；moderate 有记录和处理计划。
- [ ] 新功能或 bug fix 有对应测试，至少覆盖 happy path、错误输入、权限/边界场景。
- [ ] AI、企微、Sentry、告警、外部服务相关逻辑使用 mock 或空配置安全降级，不依赖真实密钥。
- [ ] 涉及 `server/prisma/schema.prisma`、认证、权限、审计、CI/CD、发布脚本的改动已标为高风险并请求审查。

## PR 前自审

- [ ] 使用 `docs/prompts/03-代码审查提示词.md` 做过新对话 AI 自审。
- [ ] 使用 `docs/prompts/04-测试生成提示词.md` 检查测试缺口。
- [ ] PR 描述写清需求、改动、验证、风险、上线/回滚。
- [ ] GitHub required checks 预期会跑：`lint`、`security`、`test`、`e2e-core`。
