# 活动列表内联编辑 — 测试用例

## 测试环境前置条件

- 已登录具有 `activity:update` 权限的项目经理账号
- 项目下至少有 3 条活动（包含不同类型：任务、里程碑）
- 至少有一条活动设置了前置依赖

---

## TC-01: 前置依赖（predecessor）内联编辑

### TC-01-1: 正常编辑前置依赖
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 双击「前置」列的单元格 | 出现 Input 文本框，预填当前值（如 `3FS+2`），光标聚焦 |
| 2 | 输入 `2FS+1, 3` 并按 Enter | 提交成功，显示 `2FS+1, 3`，活动日期自动更新 |

### TC-01-2: 清空前置依赖
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 双击有前置依赖的单元格 | 出现 Input 文本框 |
| 2 | 清空内容并按 Enter | 提交成功，前置列显示 `-` |

### TC-01-3: 格式错误
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 双击「前置」列 | 出现 Input |
| 2 | 输入 `abc` 并按 Enter | 提示 `格式错误，示例: 3FS+2, 5`，不提交 |

### TC-01-4: 自引用校验
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 在序号为 3 的活动上双击「前置」 | 出现 Input |
| 2 | 输入 `3FS` 并按 Enter | 提示格式错误（不允许自引用） |

### TC-01-5: 失焦提交
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 双击「前置」列，输入有效值 | 出现 Input |
| 2 | 点击表格外空白区域 | Input 消失，值已保存 |

## TC-02: 阶段（phase）内联编辑

### TC-02-1: 选择阶段
| 步骤 | 操作 | 预期结果 |

### TC-02-3: 点击空白关闭
| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 1 | 双击「阶段」列 | Select 展开 |
